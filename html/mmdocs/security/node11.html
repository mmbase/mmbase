context

This implementation can be used, when a more advanged system is needed.
  • org.mmbase.security.implementation.context.ContextAuthentication This class will load several modules, which can be used for authentication. The config file for this authentication can be found in the config directory security/context/config.xml. In this xml the modules, users, groups and contexes can be specified. Notice that the name of the context must be the same as the name of the owner field inside MMBase.

  • org.mmbase.security.implementation.context.ContextAuthorization This class will load the authorization. The config file for this authentication can be found in the config directory security/context/config.xml. In this xml the modules, users, groups and contextes can be specified. Notice that the name of the context must be the same as the name of the owner field inside MMBase.
The config file of the context security has the following settings:

  • /config The configuration for the user->group*->context->object security

  • /config/loginmodules The login modules are used for authentication of the users, the name is the method which is used during the login

  • /config/loginmodules is the module whitch should be loaded and has the following attributes:
    • name The name of the module, this will be used to lookup the modules. This is specified when a user wants to do a login.
    • class The class which should be used for a login on the system.
  • /config/loginmodules/properties OPTIONAL extra properties that can be used by the loginmodule.

  • /config/accounts This is used by the loginmodules, to get the users with their properties
  • /config/accounts/username The information about the users which are known to our system. it has the following attributes:
    • name The name name of the user.
    • context The context which should be used as default context: when a new object is created by this user, the context will be set to the value specified here.
  • /config/accounts/username/identify The information for a user for a specific login module. The value of this tag could be used by a loginmodule to verify the user.
    • type What kind of login module should use this information
    • rank What will be the rank of the user after a succesfull login
  • /config/groups The groups which are known to our system.
  • /config/groups/group A group in our system.
    • name The name of the current group
  • /config/groups/group/contains Something that gets the rights of the current group.
    • type The type of what gets the rights of the current group.
    • named The name of what gets the rights of the current group.
  • /config/contexts The contexts which are in our system.
  • /config/contexts/context A context in our system
    • name The name of the context, this is also the value which is stored/used from the owner field
  • /config/contexts/context/operation A operation which can be done on the system.
    • type The type of operation which can be done on the system (can be: create/read/write/link/delete/change context )
  • /config/contexts/context/operation/contains The rights will be granted to the following group:
    • group The name of the group which will be granted the rights.
