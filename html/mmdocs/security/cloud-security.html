Cloud Security Configuration

Introduction

Security setup inside MMBase

Inside MMbase there is a security framwork, in which different security implementations can be loaded. This is designed in such a way, that it is possible to use MMBase with the demants and needs that are required from the envouirment in which MMbase is used.

Security implementation with a Builder

For small user groups and to provide an easy way to manage the users, a security implementation has been created which uses a builder for its information(mmbaseusers).

Users

The users are defined inside the builder 'mmbaseusers'. Users can be added by the user 'admin' (default password: 'admin2k'). The passwords are stored with an MD5 encryption, and can only be changed by the user itselve, or by the 'admin'.

Rights

For every node in the system the following rules apply:
    - Every node can be read
    - Admin may do everything, except deleting it's own usernode
    - Normal users may create new nodes of the type's that are not system builders
    - Normal users may create new nodes of the type's that are not system builders

Context

Everynode has it's own context. The node can belong to a user (only that user and the admin can change it) or it is shared (all logged in users can change the node)

Configuration

In the file security.xml the cloud security has to be loaded, the builder mmbaseusers is responsible for managment of the users.

Deploment

In the file security.xml the cloud security has to be loaded, futhermore the application with name cloudsecurity has to be deployed (builder will be active).
