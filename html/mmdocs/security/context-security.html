Context Security Configuratie

Inleiding

Security opzet binnen MMBase

Binnen MMBase is een security frame work, waarin verschillende security implementaties kunnen worden gebruikt. Dit is op zo'n manier opgezet dat het mogelijk is om MMBase te gebruiken met wensen en eisen die gesteld worden door de omgeving waarin MMBase draaid.

Security implementatie met Contexten

Binnen de NOS is ervoor gekozen om een implementatie te maken waar bij gebruik word gemaakt van Contexten. Contexten zijn definities van rechten, dit wil zeggen dat elk object binnen MMBase een specifieke context heeft. Deze opzet heeft de volgende belangrijke onderdelen:

Gebruikers (users)

Groepen (users)

Contexten (contexs)

Gebruikers

De gebruikers van MMBase kunnen verschillende nivo's hebben. Er is 1 belangrijk onderscheid:

Anonieme gebruikers (deze worden via de 'anonymous' module geladen, en zijn de 'normale' pagina bezoekers)

Ingelogde gebruikers (deze worden via de 'user/password' module geladen, en zijn de gebruikers die wijzigingen op de data aanbrengen)

Ingelogde gebruikers kunnen verder nog een Rang (rank) krijgen, welke kan worden gebruikt om te kijken hoe 'gewichtig' een gebruiker is. Een anonymous gebruiker zal altijd de rang van ANONYMOUS hebben.

Groepen

De groepen van MMBase bestaan uit gebruikers en/of groepen. Op deze manier is het mogelijk om zo een afspiegeling te maken van bijvoorbeeld de bedrijfsstructuur, en zo ook de rechten te verdelen. Tevens is het mogelijk dat meerdere groepen dezelfde gebruiker bevatten.

Contexten

Elke object binnen MMBase heeft een context, dit is de naar van het veld 'owner' in een willekeurig object dat gematched word op een naam van een context. De context definieerd welke groepen de volgende rechten hebben:

Lezen (read)

Schrijven (write)

Referentie naar maken (link)

Verwijderen (delete)

Creeeren (create, geld alleen voor de objecten die een object type representeren (typedef of ook wel 'editors'))

Veranderen van rechten (change context)

Verder is er ook gedefinieerd naar welke andere contexten de context mag worden gezet (dus naar welke andere rechten de rechten mogen worden gezet)

Technieken

XML

XML staat voor "EXtensible Markup Language"

XML is een markup language

XML beschrijft data

XML gebruikt een DTD(Document Type Definition) voor de beschrijving van de datastructuur binnnen het document

XML met een DTD is ontwikkeld zodat het zelfbeschrijvend is.

XML Layout

A very simple XML could be the following:

<?xml version="1.0" encoding="utf-8"?>

<!DOCTYPE root SYSTEM "definition.dtd">

<root>

</root>

Deze XML heeft een encoding van "utf-8" verder zal de dtd 'definition.dtd' gebruikt worden om te controleren of de dtd correct is.

DTD Layout

In hoofdstuk 2.2 werd er een simpel XML document beschreven, een DTD hiervoor zou kunnen zijn:

<!ELEMENT root #PCDATA)>

Dit zou inhouden dat alleen de <root> tags zijn toegestaan. Tussen deze tags mag vrije tekst worden ingevoerd.

XPath

Met behulp van XPath kan een specifieke locatie aangeduid worden binnen de xml. Dit kan een locatie zijn tussen ingesloten door een element, of een veld van een element. Voor XPath geld in het kort:

/ is de root van het document

Om de elementen van elkaar te scheiden moet daartussen een '/' geplaatst worden

Om een veld binnen een element aan te duiden gebruikt met de volgende notatie: 'element[@veldnaam]'

Configureren van security-framework

Algemeen

Het security framework zorgt ervoor dat er verschillende implemenaties van security kunnen worden gebruikt binnen MMBase. De implemenatie die geladen word, is afhankelijk van het configuratie bestand dat zich bevind op de locatie %configdir%/security/security.xml.

Voorbeeld XML

De standaard opgeleverde %confgidir%/security/security.xml zal er ongeveer zo uitzien:

<?xml version="1.0" encoding="UTF-8" ?>

<!DOCTYPE security PUBLIC "//MMBase - security//" "http://www.mmbase.org/dtd/security.dtd">

<security active="true">

<authentication class="org.mmbase.security.implementation.context.ContextAuthentication" url="context/config.xml" />

<authorization class="org.mmbase.security.implementation.context.ContextAuthorization" url="context/config.xml" />

<sharedsecret>appelmoes</sharedsecret>

</security>

(De commentaar regels zijn hier weggelaten)

Binnnen deze XML kunnen we de volgende waardes worden veranderd:

/security[@active] : Valide waardes zijn 'true' en 'false'. Wanneer 'true' dan zal de security die in het document aangeduid word, worden geladen. Tevens zal het document van tijd tot tijd worden gecontroleerd op wijzigingen en wanneer dit het geval is opnieuw worden ingeladen. Wanneer de waarde 'false' is, zal er geen security gebruikt worden en zal het bestand niet gecontroleerd worden op wijzigingen.

/security/authentication[@class] : De implementatie die gebruikt moet worden om het inloggen te controleren.

/security/authentication[@url] : Hier is de mogelijkheid om extra informatie door te spelen aan de authenticatie implementatie(zoals de locatie van een additionele configuratie)

/security/authorization[@class] : De implementatie die gebruikt moet worden om te controleren of operaties toegestaan zijn.

/security/authorization[@url] : Hier is de mogelijkheid om extra informatie door te spelen aan de autorizatie implementatie(zoals de locatie van een additionele configuratie)

/security/sharedsecret : Een string die gebruikt word tussen verschillende instanties om met elkaar te kunnen communiceren. Dit is alleen vantoepassing wanneer er meer dan 1 instantie van MMBase draaid op dezelfde datasource.

Framework XML voor Context Security

Wanneer de volgende XML word gebruikt, word de context security geladen:

<?xml version="1.0" encoding="utf-8" ?>

<!DOCTYPE security PUBLIC "//MMBase - security//" "http://www.mmbase.org/dtd/security.dtd">

<security active="true">

<authentication class="org.mmbase.security.implementation.context.ContextAuthentication" url="context/config.xml" />

<authorization class="org.mmbase.security.implementation.context.ContextAuthorization" url="context/config.xml" />

<sharedsecret></sharedsecret>

</security>

/security[@active='true'] : De security zal actief zijn

/security/authentication[@class='org.mmbase.security.implementation.context.ContextAuthentication'] : Dit is de implementatie die de login voor de context security regelt.

/security/authentication[@url='context/config.xml'] : De locatie waar de configuratie van de context security zich bevind.

/security/authorization[@class='org.mmbase.security.implementation.context.ContextAuthorization'] : Dit is de implementatie voor de controle op operaties op de objecten binnen MMBase.

/security/authorization[@url='context/config.xml'] : De locatie waar de configuratie van de context security zich bevind.

/security/sharedsecret/ = "" : Er hoeft geen shared secret gedefinieerd te zijn, sinds MMBase niet behoeft te communiceren met andere instanties van MMBase, welke op dezelfde database source.

Framework DTD voor security

Voor de volledigheid van het document word hier de DTD van de XML die boven is vermeld gegeven:

<!ATTLIST security active (true|false) #REQUIRED >

<!ELEMENT security ( authentication, authorization, sharedsecret ) >

<!ELEMENT authentication EMPTY >

<!ATTLIST authentication class NMTOKEN #REQUIRED >

<!ATTLIST authentication url CDATA #REQUIRED >

<!ELEMENT authorization EMPTY >

<!ATTLIST authorization class NMTOKEN #REQUIRED >

<!ATTLIST authorization url CDATA #REQUIRED >

<!ELEMENT sharedsecret ( #PCDATA ) >

Werking Context Security

Referentie veld

Binnen MMBase heeft elk object een veld dat als naam heeft 'owner'. Dit veld word gebruikt door de context security om rechten aan een object toe te voegen en word gematched aan een context met dezelfde naam binnen dezelfde context. Wanneer dit niet bestaat, zal de default worden gebruikt en een waarschuwing in de logs worden gemeld.

Contexts

Een context definieert welke rechten groepen hebben op een bepaald object. Tevens is binnen de context ook gedefineerd naar welke andere contexten het object met de huidige context mag worden veranderd. Op het moment dat het ownerfield naar deze context is veranderd, dan zijn de rechten in de rechten van de betreffende context veranderd.

Groepen

Een groep is een manier om een verzameling gebruikers en/of groepen te definieren. Een groep kan bestaan uit een andere groepen, gebruikers, of een combinatie van beiden.

Gebruikers

Anonieme gebruikers

Deze worden via de 'anonymous' module geladen, en zijn de 'normale' pagina bezoekers

Een anonymous gebruiker zal altijd de rang van ANONYMOUS hebben.

Ingelogde gebruikers

Dze worden via de 'user/password' module geladen, en zijn de gebruikers die wijzigingen op de data aanbrengen

Ingelogde gebruikers hebben een Rang (rank) krijgen, welke kan worden gebruikt om te kijken hoe 'gewichtig' een gebruiker is.

Totaal plaatje


Operaties

Lezen

Om een object te mogen lezen moet een gebruiker 'read' rechten hebben.

Schrijven

Om binnen object dingen te mogen wijzigen heeft een gebruiker 'write' rechten nodig.

Verwijderen

Om een object te mogen verwijderen heeft de gebruiker 'delete' rechten nodig.

Referentie naar maken

De 'link' geeft aan dat een gebruiker relatie mag aan maken naar het specifieke object

 

Creeeren

De 'create' word alleen gebruikt voor builders, om te kijken of een specifiek soort object mag worden aangemaakt door een gebruiker. Binnen de builder 'typedef' of ook wel 'editors' bevind.

 

Veranderen van rechten

De 'change context' geeft aan een gebruiker de rechten om de huidige rechten te wijzigen in andere rechten.

Handelingen

Creeeren nieuw object

Om een object te mogen creeeren, moet de gebruiker 'create' rechten hebben op de node die het node-type representeerd binnen MMBase (typedef tabel)

Creeeren nieuwe relatie

Om een relatie mogen creeeren, moet de gebruiker 'create' rechten hebben op de node die het node-type representeerd binnen MMBase (typedef tabel), dit zal meestal de tabel 'insrel' zijn. Verder moet de gebruiker 'link' rechten hebben op beide objecten waartussen de relatie totstandkomen.

Wijzigen object

Om een object te mogen wijzigen zijn er 'change' rechten nodig op een specifiek object.

Wijzigen rechten object

Om de rechten op een object te mogen wijzigen zijn er 'change context' rechten nodig op een specifiek object.

Verwijderen object

Om een object te mogen verwijderen heeft de gebruiker 'delete' rechten nodig.

De XML

Voorbeeld XML

Aan de hand van de volgende XML zal een uitlegworden gegeven over de opzet van het context security configuratie bestand.

<?xml version="1.0" encoding="utf-8"?>

<!DOCTYPE contextconfig PUBLIC "//MMBase - contextconfig//" "http://www.mmbase.org/dtd/securitycontextconfig.dtd">

<contextconfig>

<loginmodules>

<module name="anonymous" class="org.mmbase.security.implementation.context.AnonymousLogin" />

<module name="name/password" class="org.mmbase.security.implementation.context.PasswordLogin" />

</loginmodules>

<accounts>

<user name="anonymous" context="default" />

<user name="admin" context="admin">

<identify type="name/password" rank="administrator">admin2k</identify>

</user>

<user name="foo" context="default" >

<identify type="name/password" rank="basic user">bar</identify>

</user>

</accounts>

<groups>

<group name="everyone">

<contains type="group" named="users" />

<contains type="user" named="anonymous" />

</group>

<group name="users">

<contains type="user" named="foo" />

<contains type="group" named="administrators" />

</group>

<group name="administrators">

<contains type="user" named="admin" />

</group>

</groups>

<contexts default="default">

<context name="default">

<operation type="create">

<grant group="users" />

</operation>

<operation type="read">

<grant group="everyone" />

</operation>

<operation type="write">

<grant group="users" />

</operation>

<operation type="link">

<grant group="users" />

</operation>

<operation type="delete">

<grant group="users" />

</operation>

<operation type="change context">

<grant group="administrators" />

</operation>

<possible context="default" />

</context>

</contexts>

</contextconfig>

Opbouw XML

/contextconfig/loginmodules : Dit zijn de modules die zorgdragen voor het inloggen binnen MMBase. Het wijzigen van deze instellingen zal niet nodig zijn, tenzij het explicit word vermeld, en zal daarom buiten de beschouwing van dit document worden gelaten.

/contextconfig/accounts : Dit zijn de accounts van gebruikers die mogen inloggen binnen MMBase. Binnnen deze /context/accounts is het verstandig om altijd een 'anonymous' gebruiker te hebben, omdat ander deze niet bestaat. Verder kunnen er een nieuwe gebruikers worden toegevoegd door onder deze entry de volgende xml in te voeren:

<user name="%name%" context="%defaultcontext%" >

<identify type="name/password" rank="%rank%">%password%</identify>

</user>

Hiebij kunnen de volgende waarden worden ingesteld:

user[@name] : De naam van de gebruiker.

user[@context] : De context die objecten krijgen die aangemaakt worden door de gebruiker

user/identify[type] : Het type login (refereert naar een module uit /contextconfig/loginmodules/module[@name] ), deze zal altijd "name/password" zijn.

user/identify[rank] : De rang die de gebruiker krijgt na een login. Dit zal altijd "basic user" zijn, tenzij het een administrator is, dan zal het "admin" zijn.

user/identify : De waarde ingesloten door de 2 tag's is het password voor de desbetreffende gebruiker.

/contextconfig/groups : Dit zijn de groepen die bekend zijn binnen MMBase. Een groep heeft de volgende structuur:

<group name="%name%">

<contains type="user" named="%username%" />

<contains type="user" named="%username%" />

<contains type="group" named="%groupname%" />

<contains type="group" named="%groupname%" />

</group>

Hierbij kunnen de volgende waarden worden ingesteld:

group[@name] : De naam van groep

group/contains[@type] : Het type dat deze groep beval. Validate waardes zijn: 'user' en 'group'

group/contains[@named] : De groep of gebruiker die in deze groep zit. Op het moment dat group/contains[@type='user'] dan zal er naar een gebruiker worden gezocht binnen /contextconfig/accounts/user[@name='%username']. Wanneer de waarde van group/contains[@type='group'] dan zal er naar een naar een groep worden gezocht binnen /contextconfig/groups/group[@name='%groupname%']

/contextconfig/contexts : Dit zijn de contexten die gebruikt kunnen worden binnen het context security systeem.

/contextconfig/contexts[@default] : dit veld refereerd naar een context die gebruikt moet worden op het moment dat een context wel in het ownerfield staat binnen MMBase, maar niet gevonden kan worden. Het veld dat hier staat moet een element zijn uit /contextconfig/contexts/context[@name].

Binnen deze /contextconfig/contexts kunnen nieuwe contexten worden aangemaakt op de volgende manier:

<context name="%contextname%">

<operation type="%operation%" />

<operation type="%operation%">

<grant group="%groupname%" />

<grant group="%groupname%" />

</operation>

<possible context="%contextname%" />

<possible context="%contextname%" />

</context>

context[@name] : De naam van de context

context/operation[@type] : Het type operatie waar rechten aan worden toegekend. Validate waardes zijn: 'read', 'write', 'link', 'delete', 'change context'

context/operation/grant[@group] : De groep die de betreffende rechten voor een operatie op de context krijgt. Deze waarde moet refereren naar een waarde uit /contextconfig/groups/group[@name='%groupname%'].

context/possible[@context] : Wanneer de rechten van dit object veranderd mogen worden in andere rechten, staat hier vermeld in welke andere rechten deze dan kan veranderen. De waarde die hier gespecificeerd is refereerd naar een waarde uit /contextconfig/contexts/context[@name=%contextname%].

 

De DTD

Voor de volledigheid van het document is hier de dtd ook gegeven:

<!ELEMENT contextconfig ( loginmodules, accounts, groups, contexts ) >

<!ELEMENT loginmodules ( module+ ) >

<!ELEMENT module ( property* ) >

<!ATTLIST module class NMTOKEN #REQUIRED >

<!ATTLIST module name CDATA #REQUIRED >

<!ELEMENT property ( #PCDATA ) >

<!ATTLIST property name NMTOKEN #REQUIRED >

<!ELEMENT accounts ( user+ ) >

<!ELEMENT user ( identify* ) >

<!ATTLIST user context NMTOKEN #REQUIRED >

<!ATTLIST user name NMTOKEN #REQUIRED >

<!ELEMENT identify ( #PCDATA ) >

<!ATTLIST identify rank (administrator | basic user) #REQUIRED >

<!ATTLIST identify type CDATA #REQUIRED >

<!ELEMENT groups ( group+ ) >

<!ELEMENT group ( contains+ ) >

<!ATTLIST group name NMTOKEN #REQUIRED >

<!ELEMENT contains EMPTY >

<!ATTLIST contains named NMTOKEN #REQUIRED >

<!ATTLIST contains type NMTOKEN #REQUIRED >

<!ELEMENT contexts ( context+ ) >

<!ATTLIST contexts default NMTOKEN #REQUIRED >

<!ELEMENT context ( operation*, possible* ) >

<!ATTLIST context name NMTOKEN #REQUIRED >

<!ELEMENT operation ( grant* ) >

<!ATTLIST operation type (read | write | link | delete | change context) #REQUIRED >

<!ELEMENT grant EMPTY >

<!ATTLIST grant group NMTOKEN #REQUIRED >

<!ELEMENT possible EMPTY >

<!ATTLIST possible context NMTOKEN #REQUIRED >

Aanvullende informatie

Wanneer het bestand %configdir%/security/security.xml of wel %configdir%/security/context/config.xml gewijzigd is, dan zal deze binnen een minuut worden herladen. Wanneer er zich fouten in de configuratie bevinden, dan zal dit vermeld worden in de log file. De oude configuratie blijft dan actief.
