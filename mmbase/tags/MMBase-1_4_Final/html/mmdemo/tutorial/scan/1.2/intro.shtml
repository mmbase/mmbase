1.2 Setting a object alias
In the previous example we used a object number to talk to the cloud when we wanted to display the people node for Rob but we urge you to use names instead of numbers in site starting pages and never used fixed numbers in your shtml pages (like we showed in 1.1).

This can by attaching a name to a number/node, this is similar as ip numbers vs hostnames where a www.xxx.com adress is a alias for a ip number. Just like when a ip-number is changes the hostname can stay the same we want important nodes to have a name so we can refer to it as that and not by its object number.

You can set a alias for every node you want using the editors (oalias), in this example we set a alias called Rob for the node from example 1.1. We will now repeat 1.1 but using the alias.
